# NodeSkill Quick Reference

## Domains

| Domain | Flag | What It Covers |
|--------|------|----------------|
| pattern | `--domain pattern` | Design patterns (Factory, Strategy, Middleware, DI, etc.) |
| package | `--domain package` | Package recommendations by use case |
| error | `--domain error` | Error handling patterns and strategies |
| perf | `--domain perf` | Performance optimization techniques |
| test | `--domain test` | Testing patterns (Vitest, Jest, Playwright) |
| async | `--domain async` | Async patterns (Promises, Streams, Workers) |
| typescript | `--domain typescript` | TypeScript type system patterns |
| arch | `--domain arch` | Architecture patterns (MVC, Clean, Modular) |
| idiom | `--domain idiom` | Node.js/TypeScript idioms and conventions |
| anti | `--domain anti` | Anti-patterns and common mistakes |
| tooling | `--domain tooling` | Tools and CLI utilities |
| module | `--domain module` | npm, packages, and module management |
